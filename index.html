Best Practice
What to do What to avoid
Primary The primary action on the page Using for different actions per screen
Using more than two for the same action per screen
Secondary For secondary actions on the page Try not to have more than 1-2 on the screen at any given time.
Danger For destructive actions that are hard/impossible to undo. Place them near Primary buttons
Small When there is not enough vertical space, such as within a table. Just because they’re small, doesn’t mean it’s automatically for mobile views
Label For actions that don’t require visual prominence.
For navigation inside an app 		Link to static webpages
Used within regular body text
Wide For multi step forms Using them when the regular sized buttons work just as well
